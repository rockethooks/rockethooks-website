With this release, RocketHooks gains email as a third notification channel and opens up a public GraphQL API so you can manage hooks programmatically.

## New Features

### Email Notification Channel

- **Transactional email delivery** -- receive change events as structured email notifications
- **Customizable templates** -- configure subject lines and recipient lists per hook
- **Digest mode** -- batch multiple changes into a single daily or hourly summary email

### Public GraphQL API

- **Full hook management** -- create, update, delete, and list hooks via GraphQL mutations and queries
- **Execution history queries** -- retrieve delivery logs and change diffs programmatically
- **Real-time subscriptions** -- subscribe to hook events over WebSocket for live dashboards
- **API key authentication** -- generate scoped API keys from the dashboard

### Smart Rate Limit Handling

- **Automatic detection** -- RocketHooks recognizes `429` responses and `Retry-After` headers
- **Adaptive backoff** -- polling intervals temporarily increase when rate limits are detected, then resume normal frequency
- **Rate limit alerts** -- optional notifications when a hook consistently hits rate limits

## Improvements

- **Hook templates** -- pre-built configurations for popular APIs (Salesforce, HubSpot, Shopify, GitHub)
- **Faster dashboard** -- hook list loads 60% faster with paginated API queries
- **Improved error messages** -- clearer descriptions when API authentication fails during hook creation

## Bug Fixes

- Fixed edge case where array diff detection reported false positives on empty arrays
- Resolved Slack message truncation for payloads exceeding 3000 characters
- Corrected an issue where paused hooks could briefly resume after a deployment
