This release expands RocketHooks beyond webhooks with our first additional notification channel: Slack. We have also improved polling performance and added new JSONPath comparison operators.

## New Features

### Slack Notification Channel

- **Direct Slack delivery** -- send change events to any Slack channel or DM
- **Rich message formatting** -- events are rendered with structured blocks showing old vs. new values
- **Channel routing rules** -- route different hooks to different Slack channels based on event type
- **OAuth integration** -- connect your Slack workspace with a single click

### Extended JSONPath Operators

- **Numeric thresholds** -- trigger events only when a numeric field changes by more than a specified amount
- **Array diff detection** -- detect added, removed, or reordered items in array fields
- **Nested path wildcards** -- use `$.orders[*].status` to monitor a field across all array elements

## Improvements

- **Polling performance** -- reduced average poll-to-event latency by 40% through optimized Lambda cold starts
- **Hook limits increased** -- beta accounts now support up to 50 hooks (previously 25)
- **Dashboard UX** -- added bulk actions for pausing and resuming hooks
- **Delivery retry visibility** -- retry attempts are now shown inline in the execution history

## Bug Fixes

- Fixed an issue where OAuth token refresh could silently fail on certain providers
- Resolved intermittent duplicate event detection on rapidly changing endpoints
- Corrected timezone display in the execution history for non-UTC users
